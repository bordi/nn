# Короткий звіт до лабораторної роботи №1

## Тема

Edge Computer Vision: класифікація + Grad-CAM + оптимізація

## Мета роботи

Навчитися будувати модель комп'ютерного зору для класифікації зображень, аналізувати її рішення за допомогою `Grad-CAM` та оцінювати компроміс між якістю, розміром моделі та швидкістю роботи на CPU.

## Датасет

У роботі використано датасет `Food-101`, який містить:
- 101 клас страв
- 75 750 тренувальних зображень
- 25 250 тестових зображень

Зображення організовані по класах, а готові списки `train/test` беруться з папки `meta`.

## Постановка задачі

Потрібно було:
- навчити baseline-модель класифікації зображень
- покращити її за рахунок аугментацій і fine-tuning
- побудувати `Grad-CAM` для правильних і помилкових прикладів
- оптимізувати модель через експорт у `ONNX`
- порівняти якість і швидкість роботи

## Використаний підхід

Як базову архітектуру було обрано `MobileNetV3`, оскільки вона добре підходить для задач Edge CV: має відносно невеликий розмір і швидкий inference.

Навчання виконувалось у два етапи:

1. `Baseline`
   Було заморожено backbone моделі та донавчено лише classification head.

2. `Improved`
   Було додано аугментації зображень, а також розморожено останні блоки backbone для fine-tuning.

Для зупинки навчання застосовано `early stopping` за значенням `validation loss`.

У цьому запуску для швидкої перевірки всього пайплайну використовувався debug-режим:
- `--debug-samples 1024`
- `--baseline-epochs 1`
- `--improved-epochs 1`

Тому отримані нижче значення слід трактувати як демонстраційний результат для перевірки працездатності пайплайну, а не як максимальну якість, якої можна досягти на повному `Food-101`.

## Метрики

У роботі використовувалися такі метрики:
- `accuracy`
- `macro-F1`
- `model size (MB)`
- `latency (ms)` на CPU

## Результати

### Метрики під час навчання

Під час швидкого debug-запуску було отримано такі значення:

| Етап | Train loss | Train accuracy | Validation loss | Validation accuracy | Duration |
|---|---:|---:|---:|---:|---:|
| Baseline | 4.4411 | 0.0605 | 4.1518 | 0.1563 | 4.24 s |
| Improved | 3.5941 | 0.2549 | 3.8536 | 0.1445 | 4.55 s |

### Метрики для inference

Після навчання було виконано оцінювання `baseline`, а також найкращої `improved` моделі та її `ONNX`-версії:

| Версія моделі | Accuracy | Macro-F1 | Size (MB) | Latency (ms) |
|---|---:|---:|---:|---:|
| Baseline PyTorch | 0.1547 | 0.1133 | 16.72 | - |
| Improved PyTorch | 0.1754 | 0.1399 | 16.72 | 147.81 |
| Optimized ONNX | 0.1754 | 0.1399 | 16.51 | 3.99 |

### Короткий аналіз результатів

- `baseline` і `improved` відпрацювали коректно, а весь пайплайн успішно завершився
- на скороченій вибірці модель навчається дуже швидко, але абсолютна якість залишається невисокою, що очікувано для `Food-101`
- у цьому debug-запуску `improved` етап зменшив `validation loss`, але не дав приросту `validation accuracy` порівняно з `baseline`
- це свідчить про те, що 1024 зображення і лише 1 епоха на кожному етапі достатні для перевірки коду, але замалі для сильної якості
- експорт у `ONNX` не змінив `accuracy` і `macro-F1`, але дуже суттєво зменшив `CPU latency`

У цьому запуску прискорення inference на CPU склало приблизно у `37` разів:
- `PyTorch`: `147.81 ms`
- `ONNX`: `3.99 ms`

## Аналіз Grad-CAM

У роботі було побудовано щонайменше 10 Grad-CAM прикладів:
- 5 правильних класифікацій
- 5 помилкових класифікацій

### Короткий аналіз

За згенерованими Grad-CAM прикладами видно таке:

- у правильних прикладах модель переважно фокусується на центральній частині самої страви, а не на фоні
- для `baby_back_ribs` найбільша увага припадає на поверхню ребер і характерну текстуру м'яса
- у помилковому прикладі `apple_pie -> french_toast` модель зосередилась на підсмаженій блискучій поверхні, тобто на спільній текстурі й кольорі скоринки
- серед характерних плутанин спостерігаються візуально схожі класи, зокрема `macarons -> cup_cakes`, `spaghetti_carbonara -> spaghetti_bolognese`, `beet_salad -> greek_salad`
- це показує, що модель уже вловлює сам об'єкт їжі, але на короткому debug-запуску ще недостатньо добре розрізняє близькі класи

## Висновок

У ході роботи було реалізовано повний пайплайн для задачі класифікації зображень на датасеті `Food-101`: навчання моделі, оцінювання, пояснення рішень через `Grad-CAM`, експорт у `ONNX` та benchmark. Transfer learning дозволив швидко отримати робочу модель навіть у скороченому режимі.

Головний висновок цього запуску полягає в тому, що debug-режим добре підходить для перевірки всього процесу, але не дає високої якості через малу вибірку і дуже коротке навчання. Водночас `ONNX` показав значне прискорення inference на CPU без втрати точності, що добре демонструє компроміс між якістю та ефективністю в задачах Edge CV.
